# 减脂 App · Version 2 升级计划

## 当前版本（v1，已完成）

- 单页 HTML，5 个 Tab（今天 / 动作 / 饮食 / 进度 / 更多）
- 15 个动作，含器械数据、组数重量、展开教学
- 今日饮食时间轴、补剂提醒、训练打卡
- 饮食页：手掌法 / 简餐 / 隐形热量 / 采购清单
- 进度页：体重记录 / 打卡日历 / 阶段目标
- 视频：B站 + YouTube 双链接跳转（已修复）
- 部署：GitHub Pages，随时访问，不依赖电脑

## Version 2 核心升级方向

### 1. 自制动作讲解视频

**问题**：现有平台视频质量参差，集锦多、技术讲解少，无法针对"壶铃+杠铃套装"的器械场景。

**目标**：为每个动作制作 45-60 秒专属讲解视频，格式：
```
1. 目标肌群 3D 高亮（5s）
2. 完整动作正面+侧面演示（10s）
3. 慢动作关键节点分解（15s）
4. 常见错误对比演示（10s）
5. 一句话记忆口诀（5s）
```

**制作阶段：**

| 阶段 | 内容 | 工具 |
|---|---|---|
| 脚本分镜 | 每个动作一份分镜脚本 | 枳来写 |
| 实拍素材 | 手机固定机位拍摄 | sis 自拍 |
| 3D 肌肉动画 | 解剖动画叠加 | 剪映/CapCut + 3D 素材库 |
| 上线 | 替换 B站/YouTube 链接 | Cloudflare R2 托管 |

**视频存储**：Cloudflare R2（10GB/月免费）→ `<video>` 标签直接嵌入，比 iframe 更快，支持离线缓存

**第一批优先级**（最常用 + 最易做错）：
- 壶铃摆荡
- 杠铃罗马尼亚硬拉
- 杠铃臀推
- 哑铃酒杯深蹲
- 单臂哑铃划船

---

### 2. UI 重设计（Figma → 实现）

**现有问题：**
- 颜色：紫色系偏科技感，健康/减脂类更适合暖绿/橙/米色
- 层次：信息密度高但缺乏视觉重量对比
- 今天页面：时间轴太平，无进度感

**改进方向：**
- 今天页面顶部加"今日进度环"（已摄入/目标卡路里圆环）
- 字重对比拉大：关键数字大+粗，辅助文字小+浅
- 动作卡片加肌肉部位色块示意图
- 整体色系参考：Apple Fitness+（深绿+白+暖灰）

**流程**：sis 在 Figma 出设计稿截图 → 枳按图实现，不需要导出代码
